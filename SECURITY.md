# Security Policy

This document describes how security vulnerabilities should be handled in the NovaTube project.

---

## [.] Supported Versions

Only the latest release version of NovaTube is supported for security updates.

| Version | Supported          |
| ------- | ------------------ |
| v1.x    | [+] Yes            |
| < v1.x  | [!] No             |

---

## [.] Reporting a Vulnerability

If you discover a security vulnerability, please do not open a public GitHub issue. Instead, report it directly to the maintainer.

* Send an email detailing the vulnerability to the project maintainers.
* Please include as much information as possible to help us reproduce and address the issue (e.g. system configurations, step-by-step reproduction instructions).

We aim to acknowledge receipt of vulnerability reports within 48 hours and provide a detailed timeline for fixes within 7 days.
